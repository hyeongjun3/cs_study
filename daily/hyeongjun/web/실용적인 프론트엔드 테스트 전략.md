## 프론트엔드의 입력과 출력

### 입력 데이터
DOM 이벤트 : 마우스, 키보드, 터치 등의 입력 이벤트
- 브라우저 이벤트 시뮬레이션 API 사용

라우팅/IO : URL 변경, 네트워크/로컬 파일, 로컬 스토리지/쿠키
- 브라우저 API 목킹 / E2E 테스트 도구 사용

### 출력 데이터
코드 관점 : HTML, CSS
- 검증 : 생성된 HTML, CSS 코드의 내용을 비교

사용자 관점 : 브라우저가 렌더링한 화면(픽셀 정보)
- 검증 : 브라우저가 렌더링한화면을 갭처해서 이미지로 비교
